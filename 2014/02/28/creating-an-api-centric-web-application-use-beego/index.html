Creating an API-Centric Web Application using beego

This is an archived note from the old astaxie site. It is kept as part of the technical exploration timeline.

Introduction

Planning to start working on a new web application? In this tutorial, we'll discuss how to create an API-centric web application, and explain why this is essential in today's multi-platform world.

API?

An API is the contract between a service and its consumers. For a web application, that contract usually includes routes, request and response formats, authentication rules, error semantics, and versioning expectations.

What's an API-Centric Web Application?

An API-centric web application treats the API as the primary product surface. The browser interface is one client of that API, not the only place where business logic lives.

Why go through all this trouble?

Separating the API from the client helps teams support multiple platforms, test server behavior independently, and evolve the user interface without rewriting core service logic.

Create project

In this tutorial, we'll be creating a simple TODO list application that is API-centric and one front-end client in the browser that interacts with our TODO list application.

Implementation notes

Start by defining API resources for creating, listing, updating, and deleting items. Then implement the API handlers with beego while keeping request parsing, validation, persistence, and response formatting easy to inspect and test.

Build the browser client against the API, treating it as a consumer rather than a place to hide server-side behavior.

Conclusion

API-first design makes a web application easier to extend across clients and easier to reason about as the system grows.
